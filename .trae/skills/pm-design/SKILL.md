# PM 产品设计方法论

## 需求分析框架

### 1. 功能描述
- 这个功能是什么？
- 解决什么问题？
- 用户场景是什么？

### 2. 交互设计
- 用户操作流程
- 界面元素说明
- 状态变化说明
- 边界情况处理

### 3. 技术方案
- 前端实现方案
- 后端 API 设计（如需要）
- 数据库变更（如需要）
- 第三方依赖

## 设计输出模板

```markdown
## 📋 需求N: [功能名称]

### 功能描述
[一句话描述功能]

### 用户场景
[典型使用场景]

### 交互设计
1. [操作步骤1]
2. [操作步骤2]
3. ...

### 界面元素
- [元素1]: [说明]
- [元素2]: [说明]

### 技术方案
- 前端: [技术选型和实现思路]
- 后端: [API/数据变更]
- 依赖: [需要的库]

### 验收标准
- [ ] 标准1
- [ ] 标准2
```

## Survey Platform 设计规范

### UI/UX 规范
- 使用 Element Plus 组件库
- 响应式设计（H5 + PC）
- 主色调：Element Plus 默认蓝色
- 间距：遵循 Element Plus 规范

### 功能优先级判断
1. **高频使用** > 低频使用
2. **核心流程** > 辅助功能
3. **用户价值** > 技术便利
4. **实现成本** 考虑投入产出比

### 已沉淀的设计决策
- 问卷编辑采用卡片式布局
- 题目支持折叠/展开
- 批量添加选项比逐个添加更高效
- 分享功能包含链接+二维码双渠道
